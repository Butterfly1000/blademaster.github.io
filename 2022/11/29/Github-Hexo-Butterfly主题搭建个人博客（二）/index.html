这样就全局装成功了 （建议翻墙，因为第一次装的时候没有翻墙，结果失败）

在所在的目录初始化

使用 hexo init

因为 git bash 的窗口已经在 kebingzao.github.io 这个项目中了，因此这时候不需要再指定目录，而是直接在当前目录中建（这时候可以把之前的那个旧的index删掉了）: hexo init .

报错: ~/Hexo/blademaster.github.io not empty, please run hexo init on an empty folder and then copy your files into it

因为执行前要清空文件夹内所有文件，但还有.git , 将mv .git ../.git 将.git文件先移动到外部，然后执行hexo inti .,成功后再移入mv ../.git .git。

如何新建分支上传_使用git分支保存hexo博客源码到github

困扰

然而，使用过hexo搭建个人博客的朋友们想必都会遇到博客源码保存的问题。即远程仓库只会保存hexo发布后的静态HTML文件，你的博客md源文件、主题配置等还在本地，一旦电脑磁盘坏了或者换了电脑，就无法在之前仓库的基础上继续写博客。针对此问题，有不少博主会在github上单独另起一个repository来专门保存博客源文件，每次发布完新文章，需要手动地将source/post下的md文件上传到源文件对应的repository，确实也能解决问题，但总归是麻烦。后来偶然看到聪明的网友借助git分支来巧妙地实现用同一个仓库保存静态网页和博客源码，自己试了下确实很赞，故总结之。

处理

source分支保存博客源文件

master分支保存hexo发布后的静态HTML文件

具体操作

.gitignore 这个本就有了，可以添加需要忽略的内容

推送 因为这边我是在master分支操作，所以有后续一系列问题

错误尝试:

在远程创建source分支，git fetch让本地拉取到，然后再master分支上将内容push到远程source，这边是源码。master留给deploy，当然你也可以选择其他。

报错: error: src refspec source does not match any.
因为master分支push到source分支，导致报错。

所以删除了远程分支source，本地从master分支切一个git checkout -b source,然后再push origin source。
然后将master内容都删除，再push到origin master。

删除本地master分支，切到source分支。

git branch -d master

错误: error: The branch ‘master’ is not fully merged.

git branch -D master # 强制删除

然后重新拉取master分支，并删除了index.html文件。

然后切换到source分支, 设置hexo-deployer-git,和上面一直，不过要再稳妥处理一下。

首先要安装 hexo-deployer-git

接下来在 _config.yml 修改配置，确保hexo deploy推送的是master分支，hexo目录下的_config.yml文件通常会配置deploy推送的目标地址，这个一般在最初使用hexo时，就会配置为master，不用改动：

git 查看远程地址 与 远程地址的设置

绑定 blog.qdgg17.top

新建一个 CNAME 文件，放到 source 文件夹中

sudo touch CNAME

打开写入blog.qdgg17.top

然后用 hexo d 部署到项目上。
